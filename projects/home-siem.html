Home SIEM e Relatório de Investigação

Dificuldade: ⭐⭐ 2/10 Duração estimada: 2-3 dias Estado: Em planeamento

Objetivo

Instalar e configurar um SIEM (Elastic Stack ou Microsoft Sentinel) local para ingerir logs do Windows e Sysmon, detetar ataques simulados, e produzir um write-up de investigação completo que documente o processo de deteção do início ao fim. A documentação é mais importante que a instalação.

Porquê este projeto?

Segundo o canal Mad Hat, este é um dos projetos mais valorizados para entry-level porque demonstra que sabes investigar e não apenas seguir tutoriais. O write-up é o que prova a um recrutador que tens capacidade analítica para um cargo SOC.

Ferramentas e Tecnologias

  • Elastic Stack (Elasticsearch, Kibana, Logstash) — SIEM open-source
  • Sysmon — Monitorização avançada de eventos do Windows
  • Windows Event Logs — Security, System, Application
  • Kali Linux — Para simular ataques
  • Atomic Red Team — Testes de deteção automatizados

Passos do Projeto

  1. Configurar Windows VM com logging avançado e Sysmon
  2. Instalar Elastic Stack (ou usar Elastic Cloud grátis)
  3. Ingerir logs do Windows para o SIEM via Winlogbeat
  4. Criar dashboards e alertas no Kibana
  5. Simular ataques (ex: brute force RDP, PowerShell abuse)
  6. Documentar a investigação: o que detetaste, como, e porquê
  7. Publicar o write-up completo no portfólio

Entregáveis

  • Write-up detalhado com screenshots do dashboard
  • Explicação da lógica de cada alerta
  • Cronologia do ataque: deteção → análise → conclusão
  • Link para o repositório GitHub com configurações
